Function Call 和 MCP有什么区别？Function Call 是大模型自带的能力吗？

区别先说结论
Function Call：是“大模型输出一种结构化调用意图”的能力（类似：我要调用 move_file，参数是...）。
MCP：是“让模型/Agent用统一协议去访问外部工具和资源”的标准。
所以：

Function Call 偏“调用动作格式”
MCP 偏“工具接入与通信规范”
Function Call 是不是大模型自带能力？
是，但要分层理解：

大模型“自带”的是：
生成结构化调用请求（工具名 + 参数）的能力。
大模型“不自带”的是：
真实执行这个调用的后端系统（比如文件系统、数据库、浏览器）。
也就是说，模型可以说“请调用 move_file(a,b)”，
但真正去动文件，需要你的程序/MCP/工具执行器来完成。
